# Codex Subagent 구현 데모 영상 구성안

## 영상 제목 후보

1. Codex Subagent로 작은 AI 개발팀 만들기
2. 같은 요청을 5명의 AI 역할에게 나눠 맡기면 생기는 일
3. Codex Subagent 데모: 기획자, 평가자, 개발자, 보안 점검자, 디자이너

## 영상의 한 줄 메시지

Codex subagent는 복잡한 작업을 여러 전문 역할로 나눠 병렬로 처리하고, 메인 대화는 판단과 통합에 집중하게 해주는 기능입니다.

## 추천 데모 흐름

### 1장. 문제 제시

화면에 빈 프로젝트 폴더나 간단한 요구사항을 보여줍니다.

```text
오늘은 Codex에게 앱 하나를 그냥 만들어 달라고 하지 않고,
기획자, 평가자, 개발자, 보조 개발자, 디자이너 역할로 나눠 맡겨보겠습니다.
```

### 2장. 기획자 subagent 실행

기획자에게 구현 데모 아이디어 5개를 요청합니다.

보여줄 포인트:
- 혼자서 바로 구현하지 않고 먼저 후보를 넓게 만듭니다.
- 각 아이디어에는 화면에서 무엇이 바뀌는지 포함하게 합니다.

### 3장. 평가자 subagent 실행

평가자에게 기획자의 아이디어를 냉정하게 평가하게 합니다.

평가 기준:
- 영상에서 바로 이해되는가
- 구현 시간이 너무 길지 않은가
- 결과 차이가 화면에 잘 보이는가
- 다른 프로젝트에도 재사용 가능한가
- 위험하거나 과장된 설명이 없는가

### 4장. 선택된 아이디어 발표

추천 아이디어:

```text
작업 관리 대시보드
```

이유:
- 같은 요구사항으로도 미니멀 버전, 데이터 대시보드 버전, 모바일 중심 버전처럼 차이가 잘 보입니다.
- 작업 상태 변경이 카드, 지표, 그래프에 즉시 반영되어 화면 녹화에 좋습니다.
- 업무 자동화, 프로젝트 관리, 개인 생산성 도구 등으로 확장하기 쉽습니다.

### 5장. 개발자, 보조 개발자, 디자이너 병렬 실행

세 역할을 동시에 맡깁니다.

개발자:
- 파일 구조를 제안합니다.
- 템플릿화와 저장 방식을 정리합니다.
- Git 커밋과 백업 흐름을 담당합니다.

보조 개발자:
- 테스트가 부족한 부분을 찾습니다.
- 보안 취약점과 운영 위험을 점검합니다.
- 구현 전에 막아야 할 리스크를 정리합니다.

디자이너:
- UI의 첫인상을 다듬습니다.
- 화면 녹화에서 잘 보이는 레이아웃을 제안합니다.
- 정보 우선순위와 색상 사용을 정리합니다.

### 6장. 결과 통합

메인 Codex가 subagent 결과를 모아 최종 실행 계획으로 정리합니다.

보여줄 포인트:
- 여러 결과가 난잡하게 쌓이는 것이 아니라, 메인 대화에서 최종 판단을 합니다.
- 사용자는 각 역할의 결과를 보고 승인하거나 수정할 수 있습니다.

### 7장. 실제 산출물 저장

템플릿, 가이드, agent 설정, 백업 파일, Git 커밋을 보여줍니다.

```text
이제 이 방식은 한 번짜리 데모가 아니라,
다음 프로젝트에서도 다시 꺼내 쓸 수 있는 템플릿이 됐습니다.
```

## 5분 영상용 시간 배분

- 0:00-0:30 문제 제시
- 0:30-1:20 subagent 개념 설명
- 1:20-2:00 기획자 실행
- 2:00-2:40 평가자 실행
- 2:40-3:40 개발자, 보조 개발자, 디자이너 병렬 실행
- 3:40-4:30 결과 통합과 산출물 확인
- 4:30-5:00 다른 프로젝트에 재사용하는 법

## 촬영 체크리스트

- 화면에 역할 이름이 잘 보이게 합니다.
- 각 subagent에게 맡긴 일이 서로 다르다는 점을 말로 설명합니다.
- 결과물은 긴 텍스트보다 표, 목록, 파일 구조 중심으로 보여줍니다.
- “시간 절약”보다 “판단 품질 향상”과 “역할 분담”을 강조합니다.
- 마지막에는 같은 템플릿을 다른 프로젝트에 적용할 수 있음을 보여줍니다.

# Codex Subagent 데모 키트 사용가이드

## 목적

이 폴더는 Codex subagent 기능의 장점과 사용법을 쉽게 설명하는 구현 데모 영상을 만들기 위한 템플릿입니다. 핵심 메시지는 단순합니다.

> 하나의 요청을 여러 전문 역할에게 동시에 맡기면, 더 빠르게 여러 방향을 비교하고 더 나은 결과를 고를 수 있습니다.

## 이 데모에서 보여줄 장점

1. 역할 분담이 명확합니다.
   - 기획자는 좋은 데모 아이디어를 넓게 제안합니다.
   - 평가자는 그 아이디어가 실제 영상에서 설득력 있는지 냉정하게 봅니다.
   - 개발자는 결과물을 템플릿으로 정리하고 저장, 백업, Git 작업까지 맡습니다.
   - 보조 개발자는 테스트 부족, 보안 취약점, 운영 위험을 점검합니다.
   - 디자이너는 UI가 화면에서 보기 좋고 이해되기 쉽게 다듬습니다.

2. 병렬 작업이 가능합니다.
   - 서로 독립적인 검토나 아이디어 탐색은 동시에 맡길 수 있습니다.
   - 메인 대화는 의사결정과 최종 정리에 집중할 수 있습니다.

3. 비교가 쉬워집니다.
   - 같은 요구사항을 여러 방향으로 만들게 한 뒤, 어떤 결과가 영상에서 가장 잘 보이는지 평가할 수 있습니다.

4. 재사용이 쉽습니다.
   - 이 키트는 특정 프로젝트 하나에만 묶이지 않고, 웹앱, 업무 자동화, 데이터 분석, 발표자료 제작 같은 여러 프로젝트 유형에 맞춰 바꿔 쓸 수 있습니다.

## 공식 동작 기준 요약

OpenAI Codex 문서 기준으로 Codex는 사용자가 명시적으로 요청할 때 subagent workflow를 실행합니다. Subagent는 독립적인 작업을 맡고, Codex는 여러 subagent 결과를 모아 요약할 수 있습니다.

프로젝트에 custom agent를 저장하려면 `.codex/agents/` 아래에 TOML 파일을 둘 수 있습니다. 이 파일에는 `name`, `description`, `developer_instructions`가 필요합니다.

참고:
- https://developers.openai.com/codex/subagents
- https://developers.openai.com/codex/concepts/subagents
- https://developers.openai.com/codex/guides/agents-md

## 빠른 실행 프롬프트

아래 프롬프트를 Codex에 붙여 넣으면 이 데모 흐름을 바로 실행할 수 있습니다.

```text
이 프로젝트에서 Codex subagent 데모 영상을 만들려고 합니다.

다음 역할의 subagent를 사용해주세요.
1. 기획자 서브에이전트: 구현 데모 아이디어를 제안합니다.
2. 평가자 서브에이전트: 제안된 아이디어를 냉정하게 평가합니다.
3. 개발자 서브에이전트: 선택된 아이디어를 템플릿, 가이드, 저장, 백업, Git 작업까지 이어갑니다.
4. 보조 개발자 서브에이전트: 테스트 부족, 보안 취약점, 운영 위험을 점검합니다.
5. 디자이너 서브에이전트: UI와 시각적 전달력을 다듬습니다.

먼저 기획자 서브에이전트에게 데모 아이디어 5개를 제안하게 하고,
그 다음 평가자 서브에이전트에게 영상 설득력, 구현 난이도, 화면에서의 변화, 재사용성 기준으로 평가하게 해주세요.
평가 결과 가장 좋은 아이디어 하나를 고른 뒤,
개발자 서브에이전트, 보조 개발자 서브에이전트, 디자이너 서브에이전트를 병렬로 사용해서
구현 계획, 위험 점검, UI 개선 방향을 각각 제안하게 해주세요.

마지막에는 제가 바로 촬영할 수 있도록 다음 형식으로 정리해주세요.
- 선택된 데모 아이디어
- 왜 이 아이디어가 좋은지
- 화면 녹화 순서
- 각 subagent가 맡은 일
- 최종 산출물 목록
- 다음에 다른 프로젝트에 적용하는 방법
```

## 추천 데모 주제

가장 추천하는 주제는 `작업 관리 대시보드`입니다.

이유는 화면 변화가 명확하기 때문입니다. 예를 들어 작업 상태를 옮기면 카드, 진행률, 위험 표시, 간단한 그래프가 함께 바뀝니다. 영상에서 subagent가 만든 결과의 차이를 보여주기 쉽고, 비전문가도 이해하기 좋습니다.

## 영상에서 강조할 말

```text
혼자 한 번에 다 시키는 것이 아니라,
작은 AI 팀을 만들어 각자 잘하는 일을 맡기는 방식입니다.

기획자는 넓게 보고,
평가자는 냉정하게 자르고,
개발자는 산출물로 만들고,
보조 개발자는 위험을 찾고,
디자이너는 보는 사람이 바로 이해하게 만듭니다.
```

## 주의할 점

- Subagent는 명시적으로 요청할 때 사용합니다.
- 각 subagent에게는 작고 분명한 일을 맡기는 편이 좋습니다.
- 여러 subagent가 동시에 파일을 수정하면 충돌이 날 수 있으므로, 쓰기 작업은 담당 파일을 분리하거나 한 명의 개발자가 통합하게 하세요.
- 검토나 아이디어 탐색처럼 읽기 중심 작업은 병렬 subagent에 특히 잘 맞습니다.
- 토큰 사용량은 단일 agent보다 늘어날 수 있으므로, 데모에서는 역할과 산출물 형식을 짧고 분명하게 지정하세요.

# 마스터 실행 프롬프트

아래 프롬프트는 프로젝트 유형만 바꿔서 반복 사용할 수 있는 Codex subagent 실행 템플릿입니다.

```text
이 프로젝트에서 Codex subagent를 사용해 구현 데모를 만들고 싶습니다.

프로젝트 유형:
- 여기에 프로젝트 유형을 적어주세요.
  예: 작업 관리 대시보드, 데이터 분석 리포트, 발표자료 자동 생성, 고객지원 자동화, 개인 업무 포털

목표:
- subagent 기능의 장점과 사용법을 이해하기 쉽게 보여주는 구현 데모를 만듭니다.
- 최종 결과물은 템플릿과 가이드로 재사용할 수 있어야 합니다.

사용할 subagent:
1. 기획자 서브에이전트
   - 구현 데모 아이디어를 5개 제안합니다.
   - 화면에서 변화가 잘 보이는 아이디어를 우선합니다.

2. 평가자 서브에이전트
   - 기획자 서브에이전트의 아이디어를 냉정하게 평가합니다.
   - 영상 설득력, 구현 난이도, 화면 변화, 재사용성, 위험을 기준으로 봅니다.

3. 개발자 서브에이전트
   - 선택된 아이디어를 실제 산출물로 만듭니다.
   - 템플릿, 가이드, 백업, Git 커밋과 푸시까지 맡습니다.

4. 보조 개발자 서브에이전트
   - 테스트 부족, 보안 취약점, 운영 위험을 점검합니다.
   - 공개 저장소에 올리면 안 되는 정보가 있는지 확인합니다.

5. 디자이너 서브에이전트
   - UI를 보기 좋고 이해하기 쉽게 다듬습니다.
   - 화면 녹화에서 변화가 잘 보이도록 레이아웃을 제안합니다.

진행 방식:
1. 먼저 기획자 서브에이전트를 실행해 후보 아이디어를 받습니다.
2. 평가자 서브에이전트가 후보를 평가하고 최종 후보 1개를 고릅니다.
3. 제가 최종 후보를 승인하면 개발자 서브에이전트, 보조 개발자 서브에이전트, 디자이너 서브에이전트를 병렬로 실행합니다.
4. 세 subagent의 결과를 통합해 구현 계획과 산출물 목록을 만듭니다.
5. 파일을 만들기 전에는 어떤 파일을 만들지 먼저 보여주고 제 승인을 받습니다.
6. 기존 파일 삭제, 덮어쓰기, 이름 변경은 제가 명시적으로 승인하기 전까지 하지 않습니다.
7. 새 파일은 YYYY-MM-DD-설명적이름 형식으로 만듭니다.
8. 작업이 끝나면 생성하거나 수정한 파일 목록, 백업 위치, Git 커밋과 푸시 결과를 알려주세요.

최종 출력 형식:
- 선택된 데모 아이디어
- subagent별 역할
- 구현 단계
- 위험 점검 결과
- UI 개선 방향
- 생성 파일 목록
- 백업 위치
- Git 결과
```

## 프로젝트 유형별 바꿀 부분

### 웹앱

```text
프로젝트 유형:
- 작업 관리 대시보드 웹앱

중요한 화면 변화:
- 작업 상태 변경
- 진행률 변화
- 위험 표시 변화
- 지표 카드 변화
```

### 데이터 분석

```text
프로젝트 유형:
- 엑셀 또는 CSV 데이터를 분석해 리포트를 만드는 데모

중요한 화면 변화:
- 원본 데이터 요약
- 비교표 생성
- 차트 생성
- 결론 문장 자동 작성
```

### 발표자료

```text
프로젝트 유형:
- 아이디어를 발표자료로 바꾸는 데모

중요한 화면 변화:
- 목차 생성
- 슬라이드 구조 생성
- 시각 스타일 개선
- 발표자 노트 작성
```

### 업무 자동화

```text
프로젝트 유형:
- 반복 업무를 자동화하는 데모

중요한 화면 변화:
- 입력 자료 확인
- 처리 결과 생성
- 실패 위험 점검
- 최종 보고서 저장
```
